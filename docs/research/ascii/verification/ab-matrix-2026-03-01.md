## Conclusion

The crash is deterministic to pipeline-converted XP file dimensions.
Pipeline XPs are 2x native width and height (4x cell count), causing
the WASM sprite loader to read out of bounds. The fix must ensure
pipeline output matches native XP dimensions (126x80 for player sprites).
