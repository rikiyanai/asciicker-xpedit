# 3-Doc Authority Model & Worksheet Stitch Script — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Establish a 3-doc authority model so workbench/M1/M2 truth stops fragmenting across dozens of plan and handoff files, and provide tooling to retire completed worksheets into an archive.

**Architecture:** Three canonical authority docs are the only source of truth. Everything else is either a worksheet (temporary, retire after use) or reference/archive material. A shell script automates stitching completed worksheets into a single archive doc and rewriting references repo-wide.

**Tech Stack:** Bash (stitch script), Markdown (docs), sed/grep for reference rewriting.

## Task 3: Implement the Stitch Script

**Files:**
- Create: `scripts/doc_lifecycle_stitch.sh`

**Step 1: Write the stitch script**

Requirements from handoff:
1. Append a worksheet into the archive
2. Update archive TOC/anchors
3. Delete the original worksheet
4. Rewrite path references across the repo to the archive anchor
5. Append a doc-lifecycle violation note to PLAYWRIGHT_FAILURE_LOG.md (for audit trail)

Script interface:
```bash
#!/usr/bin/env bash
# scripts/doc_lifecycle_stitch.sh
#
# Retire a worksheet into the archive.
#
# Usage:
#   bash scripts/doc_lifecycle_stitch.sh <worksheet-path> [--reason <reason>]
#   bash scripts/doc_lifecycle_stitch.sh --help
#   bash scripts/doc_lifecycle_stitch.sh --list-candidates
#
# What it does:
#   1. Validates the worksheet exists and is not a canonical/reference doc
#   2. Generates a slug anchor from the filename
#   3. Appends the worksheet content to docs/WORKBENCH_DOCS_ARCHIVE.md
#   4. Adds a TOC entry to the archive
#   5. Rewrites references to the old path across the repo (*.md files)
#   6. Appends a lifecycle note to PLAYWRIGHT_FAILURE_LOG.md
#   7. Deletes the original worksheet
#   8. Prints a summary of changes
```

Protected paths (script must refuse to stitch these):
- `PLAYWRIGHT_FAILURE_LOG.md`
- `docs/plans/2026-03-23-workbench-canonical-spec.md`
- `docs/plans/2026-03-23-m2-capability-canon-inventory.md`
- `AGENTS.md`, `CLAUDE.md`, `CODEX.md`
- `docs/INDEX.md`, `docs/AGENT_PROTOCOL.md`
- `docs/XP_EDITOR_ACCEPTANCE_CONTRACT.md`
- `docs/PNG_STRUCTURAL_BASELINE_CONTRACT.md`
- `docs/WORKBENCH_DOCS_ARCHIVE.md`

`--list-candidates` should list all `.md` files under `docs/` that are NOT in the protected list — these are worksheet candidates.
